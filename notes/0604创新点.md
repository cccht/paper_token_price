# 0604 创新点整理

## 核心判断

当前摘要提出的方向具有创新性，后续全文应围绕该摘要重组，而不是把摘要降级
为现有正文中的单平台直接定价模型。

这篇论文最值得保留的创新点是：将推理服务的时段性拥塞问题建模为
“平台--中间商--用户”的三方分层市场，并在该市场中同时处理批发定价、
中间商跨时段容量分配、零售定价和用户迁移选择。

该思路的关键不是简单加入“中间商”这个名词，而是让中间商成为真实策略主体。
平台不再直接面对所有终端用户，而是通过批发价格影响多个竞争性中间商；中间商
再根据批发价格、容量约束、时段拥塞和用户偏好制定零售价格与容量分配策略。
用户最终依据零售价格、时段偏好、迁移成本和预期QoS选择服务时段。

## 主要创新点

1. 平台--中间商--用户三方结构

   相比单平台直接定价模型，三方结构更接近推理服务市场中的转售、路由、
   代理平台和服务聚合场景。平台通过批发价调节下游中间商行为，中间商通过
   零售价和容量分配影响终端需求，用户通过跨时段迁移反向影响拥塞和利润。

2. 跨时段容量分配与零售定价联合决策

   中间商层不应只承担转发角色，而应拥有明确决策变量：
   每个中间商在各时段的服务容量分配，以及面向终端用户的零售价格。
   这样可以把“容量调度”和“价格调度”放进同一个优化框架。

3. 三层嵌套均衡的微观基础

   用户侧使用有限参与者拥塞博弈描述个体选择，并在势博弈框架下证明
   纯策略Nash均衡存在。随后使用大人口极限或熵正则化解释有限参与者均衡
   与logit随机用户均衡之间的关系。顶层则通过Stackelberg子博弈完美均衡
   描述平台、中间商和用户之间的序贯决策。

4. QoS拥塞与经济激励的耦合

   推理服务的价格机制不只影响收入，还会改变请求在时段之间的分布，进而影响
   尾延迟、有效吞吐、SLA达标率或可结算请求比例。论文应突出“价格--需求迁移
   --拥塞--QoS--利润”的反馈链路。

5. 可复现实验支撑机制效果

   数值实验应证明三层动态定价相对统一定价、平台直销、单中间商和无QoS定价
   等基线的增量价值。当前摘要中的最低QoS从0.6632提升至0.9918、利润提升
   80.51%可以保留，但指标名称应与新模型一致，例如“中间商总利润”、
   “平台批发收入”和“系统总利润”需要分别报告。

## 与已有工作的区别

现有LLM定价和推理服务研究大致可以分成几类：

1. LLM routing中的Stackelberg动态定价，重点通常是模型供应商定价和用户
   在多个模型之间的选择。
2. 多租户GPU云平台中的定价、扩容和稳定性控制，重点通常是容量扩张、
   backlog可排空性和控制安全。
3. LLM菜单定价和token机制设计，重点通常是产品版本、token预算、用户筛选
   和两部制合约。
4. 推理服务排队论和系统优化，重点通常是批处理、token长度、延迟和吞吐。

本文应明确区别于这些方向：本文不只是服务商直接给用户定价，也不只是GPU扩容
控制，而是研究推理服务中的批发--零售--用户迁移三方市场结构。中间商竞争、
跨时段容量分配和QoS拥塞反馈是本文的核心差异。

## 正文重组原则

后续全文应按照摘要承诺来调整，避免出现“摘要是三方模型，正文是两方模型”的
不一致。

需要优先重写以下部分：

1. 问题定义

   将现有“单一平台--用户”的两阶段模型改为“平台--多个中间商--用户”的
   三层序贯模型。平台发布批发价格，中间商选择容量分配和零售价格，用户选择
   中间商与时段组合。

2. 变量与目标函数

   平台层变量：每时段批发价。

   中间商层变量：每个中间商在各时段的容量分配和零售价格。

   用户层变量：用户选择的中间商--时段组合。

   平台目标：批发收入、QoS稳定性或系统利润。

   中间商目标：零售收入减批发成本、容量成本和QoS损失。

   用户目标：最大化价格、时段偏好、迁移成本和QoS共同决定的效用。

3. 均衡定义

   用户侧给出有限参与者拥塞博弈和PSNE存在性。

   大人口极限给出logit SUE或熵正则用户均衡。

   中间商侧给出竞争均衡或最优响应条件。
