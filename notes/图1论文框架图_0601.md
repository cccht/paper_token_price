请绘制一幅适用于 Nature 系列期刊论文的研究框架图，主题为：

**固定平均标价约束下 Token 服务跨时段定价：从需求响应机制到仿真与单 GPU 拥挤验证**

## 第一层：研究背景与问题

本层展示研究动机，包含 4 个横向模块：

1. **电力需求响应启发**
   - 峰谷负载不均
   - 价格表达时段稀缺性
   - 柔性负载可跨时段迁移

2. **Token 推理服务场景**
   - GPU 服务容量有限
   - 在线请求具有显著时段波动
   - 过载导致 TTFT 上升、超时增加和有效吞吐下降

3. **核心研究问题**
   - 固定平均标价约束下，平台能否迁移弹性需求？
   - 跨时段定价能否缓解峰时拥挤？
   - 显式 QoS 建模是否产生额外价值？

4. **三个研究问题**
   - RQ1：动态定价能否迁移需求并缓解拥挤？
   - RQ2：QoS 建模的增量价值何时出现或消失？
   - RQ3：主要结论是否对关键假设保持稳健？

箭头关系：

`电力需求响应启发 -> Token 推理服务场景 -> 核心研究问题 -> 三个研究问题`

## 第二层：领导者--跟随者理论模型

本层展示核心机制，包含 5 个横向模块：

1. **价格约束**
   - 时段价格向量：`p = (p_1, ..., p_T)`
   - 固定平均标价：`mean(p_t) = p_bar`
   - 价格边界：`p_min <= p_t <= p_max`
   - 可选账单保护：发布账单不超过统一定价基线

2. **平台：领导者**
   - 平台首先发布时段价格
   - 价格表达 GPU 容量的跨时段稀缺性
   - 目标是在约束下协调需求迁移

3. **用户：跟随者**
   - 刚性需求：仅在价格接近支付意愿上限时减少
   - 弹性需求：依据价格、时段偏好和迁移成本重新分配
   - 聚合响应：采用 logit 形式

4. **负载与 QoS 代理**
   - 总需求：`D_t = R_t + F_t`
   - 利用率：`u_t = D_t / G`
   - QoS 代理：`q(u_t)`
   - 容量紧张时，QoS 因子下降

5. **平台目标**
   - 有效结算收入
   - 单位服务成本
   - 刚性用户流失成本
   - QoS 退化成本
   - 模型内福利组成

箭头关系：

`价格约束 -> 平台发布时段价格 -> 用户聚合响应 -> 负载与 QoS 代理 -> 平台目标`

增加一条反馈箭头：

`平台目标 -> 优化更新价格 -> 平台发布时段价格`

## 第三层：理论性质与数值求解

本层展示模型可计算性，包含 3 个横向模块：

1. **理论性质**
   - 价格可行域非空条件
   - 有界均值投影的欧氏最优性质
   - 平台最优解存在性

2. **约束投影**
   - 对原始价格向量执行有界均值投影
   - 使用二分搜索确定平移参数
   - 保证价格边界与平均标价约束同时成立

3. **数值求解**
   - 多起点 SLSQP 搜索
   - 对候选价格再次投影
   - 显式记录利润、福利、QoS、约束残差和求解器状态
   - 返回经过约束校验的最佳可行候选
   - 不声称获得非凸问题的全局最优证书

箭头关系：

`理论性质 -> 约束投影 -> 多起点 SLSQP 求解 -> 最佳可行候选`

## 第四层：合成实验与主要结果

本层展示三个研究问题的实验回答，包含 4 个横向模块：

1. **策略比较**
   - 统一定价
   - 短视动态定价：搜索时忽略 QoS 退化
   - QoS 感知动态定价：搜索和评价均考虑 QoS 退化

2. **RQ1：缓解峰时拥挤**
   - QoS 感知动态定价迁移弹性需求
   - 相对短视动态定价，利润提高 `5.22%`
   - 相对统一定价，利润提高 `80.51%`
   - 最低 QoS 从 `0.6632` 提高至 `0.9918`

3. **RQ2：QoS 建模的增量价值**
   - 动态定价本身可因需求异质性产生收益
   - QoS 建模的额外价值随退化强度增加
   - 容量充足时，QoS 建模的额外价值接近消失

4. **RQ3：稳健性与边界**
   - 固定平均标价
   - 账单保护
   - 容量变化
   - 市场扩张
   - QoS 反馈
   - 规则基线、计费消融和扩展性检查

箭头关系：

`策略比较 -> RQ1 基准结果 -> RQ2 增量价值 -> RQ3 稳健性与边界`

## 第五层：单 GPU 系统测量

本层展示真实后端测量对建模动机的支持，包含 3 个横向模块：

1. **真实后端压力测试**
   - 单 GPU：NVIDIA RTX 4090
   - 后端：Ollama 与 vLLM
   - 模型：Qwen2.5-0.5B-Instruct 与 Qwen2.5-3B-Instruct
   - 指标：TTFT、吞吐、完成率、GPU 利用率、显存和 SLA 达标率

2. **拥挤现象**
   - 并发请求增加后，TTFT 明显上升
   - 高并发区 SLA 达标率下降
   - 有限批次多波次压力测试显示排队累积
   - 长上下文进一步降低吞吐并提高 P95 TTFT

3. **经验 QoS 启发式映射**
   - 以 TTFT SLA 达标率作为经验 QoS 观测
   - 构造参数迁移场景并重新优化价格
   - 用于稳健性检查
   - 不替代真实生产轨迹上的联合校准

箭头关系：

`真实后端压力测试 -> 拥挤现象 -> 经验 QoS 启发式映射 -> 合成经济模型稳健性检查`

## 第六层：结论与展望

本层作为底部总结区，包含 2 个横向模块：

1. **核心结论**
   - 约束动态定价能够迁移弹性请求并缓解峰时拥挤
   - 显式 QoS 建模在容量紧张时更有价值
   - 固定平均标价不等同于账单中性
   - 账单保护下，利润仍相对统一定价提高 `21.29%`

2. **解释边界与后续工作**
   - 当前结论限定于单平台、合成需求和参数化 QoS 代理
   - 单 GPU 微基准用于验证拥挤现象，不用于声明生产校准
   - 后续增加真实生产轨迹、多模型规模、多后端和多 GPU 验证
   - 进一步研究多平台竞争、个体级账单保护与分配公平性

箭头关系：

`合成实验结果 + 单 GPU 系统测量 -> 核心结论 -> 解释边界与后续工作`

## 整体视觉规范

- 图形类型：`schematic-led composite` 研究框架图。
- 阅读方向：自上而下，每层内部自左向右。
- 背景：纯白色。
- 字体：Sans-serif，优先使用 Arial、Helvetica；中文使用兼容的无衬线字体。
- 主色：深蓝色 `#0F4D92`。
- 辅助色：中蓝色 `#3775BA`、蓝灰色 `#5B6B7A`、低饱和青色 `#42949E`。
- 强调色：仅在关键结论模块使用低饱和绿色 `#2E7D4A`。
- 模块底色：浅蓝灰、浅青灰、浅暖灰，避免高饱和颜色。
- 线条：统一使用细线圆角矩形和简洁实线箭头。
- 禁止使用：彩虹配色、复杂图标、渐变、阴影、立体效果、大段正文。
- 每个模块标题使用短语，正文控制在 2--4 行。
- 核心公式只保留最小必要形式，不展示完整推导。
- 重点突出第二层“理论模型”和第四层“主要结果”。
- 输出尺寸建议：Nature 双栏宽度约 `183 mm`，高度不超过 `170 mm`。
- 输出格式：优先 SVG，同时导出 PDF 与 300 dpi 以上 PNG 预览。
