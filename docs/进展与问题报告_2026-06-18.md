## 二、原论文审稿发现的两个硬缺陷（已修复）

### 缺陷 M1：需求随中间商数量 J 放大
- **问题**：刚性/重放需求项对每个中间商都乘以"整个时段的总选择份额"，导致市场里中间商越多、凭空出现的需求越多。这与"刚性到达=外生请求量"的物理含义矛盾。
- **量化**：J=3 的总需求是 J=1 的 2.19 倍（独立复算坐实）。
- **后果**：原论文头条"系统利润 9525 / +93.92%"中约有 ~3 倍是放大假象；"中间商越多利润越高"的结论是 bug 驱动的。
- **修复**：改为按渠道自身份额分配，J=1 时与原式一致，每时段刚性总量与 J 无关。

### 缺陷 M3：缺少用户外部退出选项
- **问题**：用户选择在全部内部选项上归一化，没有"不购买"选项。导致利润按"零退出"计算，而论文又同时报告"退出概率 62.69%"——两者是互斥模型，不能并存。
- **修复**：加入显式 no-purchase 外部选项，使利润、inclusive value、退出概率在同一模型内自洽。

### 修复后的诚实结果（全部重跑）
- 原"QoS 感知 +13.61%"在原始需求水平降到 **+0.32%**（市场本就不拥塞）。
- QoS 感知的价值是**条件性的**：需求规模 ρ≥1.3 拥塞时才恢复 +9~14% 真实增益。
- "收入最大化摧毁用户福利"(IV 2.10→−0.52) 是缺退出选项的假象；自洽模型里价格保护是干净双赢。
- **复现验证**：修正代码在关闭所有修复开关时，逐位重现原始数字（4911.97/8383.79/9525.11），证明重写忠实。

## 三、新研究线：多厂家削峰填谷动态定价

### 3.1 模型设定（经多轮 brainstorming 确定）
- **核心问题**：GPU 算力基本固定时，能否用分时动态定价（高峰↑凌晨↓）把时间弹性负载从拥塞高峰迁到空闲低谷，不扩容就既榨满算力又保住 QoS。
- **结构**：2 个异质厂家（大 A 容量 1500 / 小 B 容量 600，部署同款开源模型如 GLM）+ 1 个可路由中间商 + 两类用户（时间刚性 / 时间弹性）+ 外部退出选项。
- **5 条可证伪预测**：P1 弹性迁移>刚性；P2 削峰；P3 保 QoS；P4 小厂更依赖动态定价；P5 增益随弹性占比上升。

### 3.2 实现与测试
- 三个模块（config / market / equilibrium），17 个单元测试全过。
- 求解架构：低维形状参数化（每厂家 4 标量）+ M=2 厂家 Nash。

### 3.3 实验结果（分区间，全部真实重跑）

**非拥塞区间（基线容量，峰值利用率仅 0.21–0.26）**
| 指标 | uniform | dynamic |
|---|---|---|
| 系统利润 | 1193 | 1139 |
| P1 迁移 | 弹性 +0.177 > 刚性 +0.077 ✓ | |
| 弹性占比扫描增益 | 0% / −0.32% / −4.51% / −3.76% / −3.00% | |

→ 非拥塞下市场不拥塞、QoS≡1，动态定价无用武之地，利润中性到微负。**P1 稳健成立。**

**拥塞区间（容量收紧到 300/120，QoS 真正退化）—— 用 fictitious play 求解**
| 指标 | uniform | dynamic(粗网格) | dynamic(细网格,未收敛) |
|---|---|---|---|
| 系统利润 | 1783 | 1949（+9.3%） | 1749（**−1.9%**） |
| 峰值利用率 | 0.782 | 0.706 | 0.666 |
| 最低 QoS | 0.756 | 0.968 | 0.990 |
| δ_A / δ_B | — | 0.383 / 0.023 | 0.059 / 0.278 |

### 3.4 对抗性鲁棒性检验的最终判定
- ✅ **P2 削峰 + P3 保 QoS 稳健**（两个网格都成立）——唯一分辨率无关的结论。
- ❌ **利润增益不稳健**：粗网格 +9.3%，细网格 −1.9%，**符号翻转** → 实为利润近中性。
- ❌ **P4 不稳健**：δ_A 与 δ_B 大小关系随网格翻转 → 必须撤掉。
- P1 在拥塞均衡处抹平（非拥塞区稳健）。

**最终可发表命题（窄而诚实）**：分时动态定价是推理服务拥塞时**可靠的 QoS 保护工具，利润大致中性——不是提利润机制**。

## 四、遇到的核心问题（按重要性）

### 问题 1【最关键·方法层】均衡求解器的收敛性，是这个课题反复栽跟头的命门
- 前两版论文都死在"平台层 K=8 随机候选不收敛，跨种子波动 ±9%"。
- 本轮新模型一开始也遇到同样问题：
  - 朴素 best-response → **极限环振荡**（sweep 间变化 0.04→0.27→0.08→0.28，不收敛）。
  - 换 Powell 优化器 → 拥塞区间单次求解 >115s 超时，且结论是优化器假象。
- **最终解法**：① 低维形状参数化（16 维→4 标量）；② 联合固定点（QoS+路由一次迭代，不嵌套）；③ 网格搜索替代 Powell（对非光滑鲁棒、成本可预测）；④ **fictitious play**（对手历史平均最优响应，阻尼极限环）+ regret 收敛判据 + 断点续跑。
- **结果**：非拥塞区间完全确定性（跨种子 rel_std=0）；拥塞区间 FP 单调收敛（regret 133→4.98）。**收敛性命门解决了。**

### 问题 2【结论稳健性】"好结果"反复被更严格的检验推翻
本轮结论翻了四次，每次都是被更严格检验推翻，这是科学诚实的体现，但也说明**这个模型的定量结论对方法选择极敏感**：
1. 朴素求解器 → 权衡（利润 −3.6%）
2. 精细朴素 → 极限环，不可信
3. FP 收敛 → 双赢（+9.3%）
4. 细网格 FP → 利润近中性（−1.9%，且未收敛），只有 QoS 保护稳健

→ **含义**：凡涉及利润绝对值/方向的 claim 都脆弱；只有定性的 QoS 保护方向稳健。论文叙事必须建立在后者上。

### 问题 3【外部效度】全合成参数 + 模型对参数敏感
- 所有经济参数（价格敏感度、迁移成本、容量成本、弹性人口占比）都是合成的，无真实校准。
- 加上问题 2，意味着**任何具体百分比数字都不可作为生产预测**，只能作机制方向说明。

### 问题 4【工程/环境】算力与运行环境限制拖慢迭代
- 单次拥塞 Nash 求解 60–200s，实验矩阵跑到半小时级。
- 运行环境会 reap 长时间后台任务（~5 分钟窗口），且 subagent 服务一度 429 不可用。
- 应对：断点续跑机制（每轮存 checkpoint，被 reap 后从断点恢复）。但这严重拖慢了精细收敛（细网格 FP 需多次手动续跑）。
- **细网格 FP 实际未收敛**（跑满 40 轮，regret 在约第 10 轮触底 ~14 后漂移升到 ~22，远未达 <5），所以细网格 −1.9% 是未收敛快照、不可作定量结论；粗网格 regret 4.98 部分是粗粒度假象——但已足够证明"利润方向不稳健"。拥塞均衡的可靠求解（更强平均化 / 变分不等式重构）是必要后续工作。

### 问题 5【P4 假设错误】预设的"小厂更依赖动态定价"不成立
- 不仅不成立，方向还随网格翻转。预设假设缺乏理论支撑，应撤掉。

## 五、当前可发表性评估（我的判断）

**诚实结论**：核心方法贡献（多厂家均衡的可复现求解 + 削峰填谷机制）**成立且经过检验**，但科学卖点比预期弱。

- **能立住的**：① 收敛性问题解决（前两版命门）；② 分时定价作为 QoS 保护机制（拥塞时削峰、保 QoS，分辨率无关）；③ P1 弹性迁移（非拥塞区稳健）。
- **立不住的**：① 利润提升（方向随方法翻转）；② P4（方向不稳健）；③ 任何具体百分比。

**适配的发表层级**：
- 按当前诚实状态，更像是**机制研究 / 方法论贡献的中等档期刊或会议**，而非"动态定价双赢"那种强卖点的顶刊正刊。
- 若要冲顶刊，需要一个更强、更反直觉、且经得起鲁棒性检验的核心发现——目前没有。
