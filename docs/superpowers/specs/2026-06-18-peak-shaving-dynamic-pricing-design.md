# 设计文档：推理服务削峰填谷动态定价（异质两厂家 + 中间商）

- **日期**：2026-06-18
- **状态**：已与用户对齐，待评审
- **关系**：本项目是 `token_dynamic_pricing_game` 系列的第三次重做。复用上一版（2026-06-18 修订版）修正后的需求口径、外部退出选项、QoS 多形态与跨种子诊断；叙事重心从"三层博弈框架（AINF-CRS）"转为"固定算力下的削峰填谷动态定价机制"。

---

## 1. 核心命题

给定**基本固定**的 GPU 算力（可有小幅波动，但最大容量不显著变化），厂家能否通过**分时动态定价**（高峰提价、凌晨降价）诱导**时间弹性负载**从拥塞高峰迁移到空闲低谷，从而在**不扩容**的前提下同时：

1. 提升算力利用率（填谷）；
2. 缓解高峰拥塞与卡顿（削峰）；
3. 保护时间刚性用户的 QoS；
4. 提升整体可服务量与各主体利润。

**辅助命题（竞争作为背景约束）**：两家部署同款开源模型（如 GLM）、服务近似同质的厂家共存，加上中间商可在两家间路由、用户可直连任一厂家官方 API，这三者共同为"高峰能提价多少"设定了一个现实天花板——提价过高，需求被分流到对手或退出。

**关键可证伪预测**（实验须正面检验，可被数据推翻）：
- P1：动态定价下，**弹性**需求的时段重心从高峰显著后移到低谷；**刚性**需求基本不动。
- P2：高峰利用率下降、低谷利用率上升（削峰填谷）。
- P3：刚性用户的 QoS 改善。
- P4：小算力厂家 B 的动态价差 $\max_t w_{B,t}-\min_t w_{B,t}$ 大于大算力厂家 A（小厂更依赖动态定价削峰）。
- P5：动态定价的系统收益随弹性用户人口占比单调上升；弹性占比→0 时收益→0。

---

## 2. 主体与结构

一天划分为 $T=8$ 个时段（$h=3$ 小时）。

| 主体 | 决策变量 | 角色 |
|---|---|---|
| 厂家 A（大算力 $G_A$） | 分时批发价 $w_{A,t}$、直连零售价 $p^D_{A,t}$ | 老牌、卡多、定价压力小 |
| 厂家 B（小算力 $G_B<G_A$） | 分时批发价 $w_{B,t}$、直连零售价 $p^D_{B,t}$ | 新入场、易被打爆、更依赖动态定价 |
| 中间商（单个） | 分时零售价 $p_t$、路由权重 $r_{m,t}$（$\sum_m r_{m,t}=1$） | 聚合两家、对用户黑箱路由 |
| 用户（两类） | 在选项集上 logit 选择 | 见下 |

**服务同质性**：两厂家部署同款开源模型，服务质量近似同质，竞争落在**价格 + 算力充裕度（QoS）**，而非模型能力差异。这是竞争的现实微观基础（GLM 开源 → 多家可部署同款）。

**用户两类**：
- **时间刚性 R**：上班族、交互式聊天、实时 agent。原生时段偏好集中在下午高峰，迁移成本 $c_s^R$ 高，价格敏感 $\alpha_R$ 低 → 锁在高峰、几乎不迁移。
- **时间弹性 E**：批量任务、离线评测、夜间数据处理、定时工作流。迁移成本 $c_s^E$ 低，价格敏感 $\alpha_E$ 高 → 被凌晨低价吸引迁移。

**用户选择集合**（每时段）：中间商聚合通道（看到零售价 $p_t$，底层路由不可见）+ 厂家 A 官方直连（看到 $p^D_{A,t}$）+ 厂家 B 官方直连（看到 $p^D_{B,t}$）+ 退出选项。跨 $T$ 时段。总选项数 $(1+M)T+1 = 3T+1 = 25$（$M=2$）。

## 3. 数学模型

### 3.1 两类用户 logit 效用

对类型 $\theta\in\{R,E\}$、选项 $k$（通道 × 时段）：
$$z^\theta_{k} = \log b_k - \alpha_\theta(\text{price}_k - p_0) - c_s^\theta\,\mathbf{1}[t\neq\tau^\theta] - \gamma(1-q_{k})$$

类型差异完全由参数承载：刚性 $(c_s^R\uparrow,\alpha_R\downarrow,\tau^R=\text{高峰})$；弹性 $(c_s^E\downarrow,\alpha_E\uparrow)$。退出选项确定效用 $u_0$（沿用上一版修正，保证利润与退出概率在同一模型内自洽）。

每类各自归一化（含退出项在分母）得份额 $s^\theta_k$；总需求为两类人口加权之和，人口权重为 $(\pi_R, \pi_E)$，$\pi_R+\pi_E=1$。

### 3.2 需求（修正口径，无 $J$ 放大）

每时段每通道需求按**渠道自身份额** $s^\theta_k$ 分配（不再乘整段总份额 $\sum_k s_k$）。这是上一版已验证的修正，必须保留。

### 3.3 厂家有效产出与 QoS

厂家 $m$ 在时段 $t$ 的总负载 = 直连需求 + 中间商路由来的需求：
$$L_{m,t} = D^{D}_{m,t} + r_{m,t}\,D^{I}_{t},\qquad u_{m,t}=L_{m,t}/G_m,\qquad q_{m,t}=\qos(u_{m,t}).$$
$G_m$ 基本固定（主模型）。算力异质 $G_B<G_A$ → 同负载下 B 更易越阈拥塞。$\qos$ 主用阈值型 $\exp[-\kappa(u-\bar u)^2]$，并做线性/sigmoid/sqrt 平滑形态稳健性（上一版基建已有）。

中间商路由为**纯 logit 软分配**（不设硬容量上限），拥塞只通过 QoS 退化体现——与现有可微固定点机制一致，利于收敛。

### 3.4 厂家利润（含闲置成本）

$$\Pi_m = \sum_t w_{m,t}\,r_{m,t}D^I_t\,q_{m,t}h + \sum_t p^D_{m,t}D^D_{m,t}q_{m,t}h - \sum_t c_g G_m h - \text{QoS退化成本}.$$
**闲置成本 $c_g G_m$ 按固定算力计**（无论是否用满都付），这是厂家有动机降凌晨价填谷的根源，也补上了上一版"容量是常数、无闲置成本"的硬伤。

中间商利润 = 零售收入 − 批发成本 − QoS 退化成本（结构同上一版中间商）。

## 4. 均衡计算与收敛架构

### 4.1 嵌套层次（逆向求解）

```
外层：2 厂家 Nash 竞争（同时定 w_{m,t}, p^D_{m,t}）
  └─ 中层：中间商最优响应（零售价 p_t + 路由 r_{m,t}）
       └─ 内层：两类用户 logit × QoS 拥塞固定点（阻尼迭代，已验证收敛）
```

### 4.2 防"第三次栽在收敛性"的三道保险

前两版死于平台层在 8–16 维连续价格空间撒 $K=8$ 随机点排序，必然不收敛、跨种子波动 ±9%。本版用三道保险根除：

1. **低维形状参数化**：分时定价不在 $2T=16$ 维自由搜索，而参数化为少数标量形状，例如
$$w_{m,t}=\bar w_m\,\bigl(1+\delta_m\,\widehat{\text{load}}_t\bigr),$$
只搜基准价 $\bar w_m$ 与动态强度 $\delta_m$。直连价 $p^D_{m,t}=\bar p^D_m(1+\delta^D_m\widehat{\text{load}}_t)$ 同构，独立的 $(\bar p^D_m,\delta^D_m)$。因此每厂家策略为 4 个标量 $(\bar w_m,\delta_m,\bar p^D_m,\delta^D_m)$（若令批发与直连共享同一动态强度则为 3 个）。$\widehat{\text{load}}_t$ 为标准化的时段负载形状（由刚性需求基线归一化得到，外生已知）。主模型采用 4 标量版本。
2. **$M=2$ best-response 求交**：对厂家 A 给定 B 做最优响应、反之亦然，画出二维 best-response 映射并数值求交点（Nash）。异质 $G_A\neq G_B$ 打破对称，通常均衡唯一，避免对称博弈的多重均衡麻烦。
3. **强制收敛诊断**：报告 best-response 迭代的策略变化范数曲线 + $\geq10$ 个随机种子的均衡均值±标准差。所有主表数字带误差棒，杜绝单种子伪精度。

### 4.3 主张边界

低维参数化是为可收敛/可复现付出的建模约束；论文须明确：结论是"在该参数化形状族内"的最优动态定价，不声称 $16$ 维连续空间的全局最优。这与诚实定位一致。

## 5. 实验设计

**主实验（核心证据）**
1. **统一定价 vs 动态定价**主表：利用率削峰填谷幅度、刚性/弹性需求时段重心位移、刚性用户 QoS 改善、系统可服务量、各主体利润。
2. **弹性负载迁移可视化**（"钱图"）：弹性需求重心从高峰后移到低谷，刚性需求基本不动。

**机制 / 异质实验**
3. **小厂 vs 大厂动态强度**：验证 P4（$\delta_B>\delta_A$）。
4. **弹性人口占比扫描**：$\pi_E$ 从 0→高，动态定价收益应单调上升（验证 P5）。
5. **竞争天花板**：有/无对手、有/无中间商路由下，高峰提价上限如何被约束。

**稳健性（复用上一版基建）**
6. QoS 函数形态（阈值/线性/sigmoid/sqrt）、退出效用 $u_0$ 敏感性、$\geq10$ 种子均值±标准差、best-response 收敛曲线。所有稳健性实验须在**拥塞被实际激活的参数区间**运行（上一版教训：在非拥塞区间做形态稳健性是无效的）。

**论文骨架**
引言（削峰填谷动机）→ 相关工作（推理服务系统 + demand response + 收益管理）→ 模型（两类用户/异质厂家/QoS/闲置成本）→ 均衡计算（低维参数化 + 收敛保证）→ 实验 1–5 → 稳健性 → 局限与未来（容量内生、多模型异质、真实 trace 校准）。

## 6. 模块与代码结构

新建独立模块，不改动既有 `fixed_market.py`（保留上一版可复现性）：

- `pricing_sim/peak_shaving_market.py`：两类用户需求、异质厂家 QoS/利润、中间商路由、含退出选项的固定点求解。复用 `fixed_market.py` 的修正口径与退出选项逻辑。
- `pricing_sim/peak_shaving_equilibrium.py`：低维形状参数化、$M=2$ best-response 求交、跨种子诊断。
- `experiments/run_peak_shaving_experiments.py`：主实验 1–5。
- `experiments/run_peak_shaving_robustness.py`：稳健性 6。
- 工件输出：`artifacts/peak_shaving/<date>/`，每个论文数字对应一条落盘 JSON 记录。

**复现纪律**（沿用上一版成功经验）：
- 关闭动态定价（$\delta_m=0$，统一价）时，结果应退化为可解释的静态基线，作为内部一致性检查。
- 每个数字可由脚本重跑；主表带跨种子均值±标准差。

## 8. 风险与未决

- **R1**：若弹性用户即便大幅降价也不足以产生可观迁移（参数依赖），P1/P5 可能很弱。缓解：$\pi_E$ 与 $\alpha_E,c_s^E$ 扫描划定机制生效的参数区间，作为诚实边界而非掩盖。
- **R2**：best-response 求交可能不收敛或多重均衡。缓解：低维 + 异质打破对称；若仍多重，报告全部交点而非择优。
- **R3**：闲置成本 $c_g$、弹性人口占比等仍为合成参数，外部效度受限。缓解：明确定位为机制仿真 + 留 trace 校准为 future work。
- **未决**：是否需要"批量 API 半价"这类现实合约形态作为弹性用户的显式锚点（可在引言引用真实做法增强动机，不必进模型）。
