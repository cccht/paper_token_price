# Token推理服务动态定价博弈论文设计

## 目标

写一篇20页纯token市场动态定价论文，零电力市场交叉。核心故事：平台通过动态定价引导弹性用户错峰，实现峰时不降智+谷时更便宜的双赢。

## 核心机制

```
平台定价 p = (p_1, ..., p_T)
    ↓
刚性用户：需求固定，不受价格影响，但受退化函数影响
弹性用户：Nash均衡选择时段（考虑价格+偏好+转移成本）
    ↓
总需求 → 利用率 u_t → 退化函数 φ(u_t)
    ↓
平台利润 = Σ_t (p_t - w) × φ(u_t) × D_t(p_t)
社会福利 = 消费者剩余 + 平台利润
```

## 论文结构（7节，~20页）

### Section 1: 引言（2页）
- token推理市场的峰谷问题
- 降智现象：利用率超阈值后推理质量急剧下降
- 弹性用户的存在：API调用可以推迟到非高峰
- 贡献列表（3-4条）

### Section 2: 相关工作（2页）
- token定价文献（Liu, Xing, Guo, Zhong, Yan, Cunningham）
- 需求响应与动态定价（一般性文献，非电力市场专属）
- 服务质量退化（LLM推理延迟/质量与负载的关系）

### Section 3: 模型（3页）
- 3.1 时段与需求结构
  - T个时段，刚性需求 D_r(t)，弹性用户集合
  - 弹性用户的时段偏好 τ_t（白天高、夜间低）
- 3.2 退化函数
  - φ(u) = exp(-κ(u - ū)²), u > ū 时退化
  - 退化对有效吞吐量的影响：有效收入 = p × φ(u)
- 3.3 用户行为模型
  - 刚性用户：需求固定，不响应价格
  - 弹性用户：选择时段 t 最大化效用 u_t = v_t - p_t + τ_t - c_shift × 1{t ≠ preferred}
  - Nash均衡：给定价格，弹性用户的时段选择构成Nash均衡
- 3.4 平台利润与社会福利
  - 利润 Π = Σ_t (p_t - w) × φ(u_t) × (D_r(t) + D_f(t))
  - 福利 W = 消费者剩余 + Π
  - 收入中性约束：平均价格 = p_cap

### Section 4: 均衡分析（3页）
- 4.1 Nash均衡存在性与唯一性
  - 弹性用户博弈是势博弈 → Nash存在
  - 条件下的唯一性（logit选择→唯一混合策略均衡）
- 4.2 最优定价结构
  - 命题：收入中性下，最优定价的峰谷价差随退化强度κ递增
  - 命题：退化感知定价的利润 ≥ 统一定价（收入中性下）
- 4.3 Stackelberg扩展
  - 平台先定价（领导者），用户Nash响应（跟随者）
  - 均衡价格的结构特征

### Section 5: 机制设计（3页）
- 5.1 信息不对称：平台观测不到用户类型（刚性/弹性）
- 5.2 激励相容条件
  - 弹性用户自选择谷时段（IC约束）
  - 刚性用户不伪装为弹性（IR约束）
- 5.3 最优价格菜单
  - 命题：退化感知下的最优价格菜单结构
  - 与统一定价的对比

### Section 6: 数值仿真（4页）
- 6.1 参数设定
  - 复用现有实验参数：G=950, w=$0.40, WTP=$1.80, κ=15, ū=0.82
- 6.2 收入中性动态定价 vs 统一定价
  - 利润 +82.7%, 福利 +9.4%, φ 0.663→0.988
  - 弹性需求峰时 -62%, 谷时 +442%
- 6.3 退化惩罚与短视定价
  - 短视定价：利润仅高2.2%但φ处于退化边缘
- 6.4 价格上限敏感性
  - p_cap = $0.60~$1.20的扫描
- 6.5 福利分解
  - 利润增量 vs 消费者剩余变化 vs 退化避免收益

### Section 7: 讨论+结论（3页）
- 7.1 退化感知定价的实践启示
- 7.2 价格上限的监管含义
- 7.3 局限性
- 7.4 未来方向
- 7.5 结论
