## 1. 核心论点（一句话）

在算力固定、需求总量基本固定的推理服务市场中，分时动态定价主要**重新分配**剩余（从拥塞损失转向服务质量与用户体验），而非**放大**厂家利润。因此在拥塞区间观察到的稳健结论是"QoS 保护"，而利润效应近似中性——这与"动态定价/surge pricing 是厂家牟利工具"的通常预期相反。

---

## 2. 利润为何打平：剩余重分配而非创造

设固定算力 $G_m$、近似固定的总需求。动态定价（高峰提价 $w_t\uparrow$、低谷降价 $w_t\downarrow$）只在时间维度上重排需求，不创造新算力或新需求。其对厂家利润的影响可分解为方向相反的两组力量。

**正向（提升利润）：**
\begin{itemize}
  \item \textbf{高峰提价提高单位毛利}：拥塞时段每单位有效服务的毛利上升。
  \item \textbf{填谷利用闲置算力}：低谷降价吸引时间弹性负载，原本空转仍计闲置成本 $c_g G_m$ 的 GPU 产生额外可结算量。
  \item \textbf{削峰使 QoS 回升、有效结算量增加}：利用率从越过退化阈值回落到阈值以下，完成率 $q_{m,t}$ 上升，原本因拥塞折损的请求恢复计费。
\end{itemize}

**负向（压低利润）：**
\begin{itemize}
  \item \textbf{高峰提价驱离需求}：价格敏感用户在高峰流向竞争厂家、直连通道或退出，形成高峰需求损失。
  \item \textbf{低谷降价摊薄单位毛利}：填谷吸引来的量以低价换取，单位毛利下降。
  \item \textbf{被挤出的弹性用户部分净退出}：经外部退出选项离开市场的需求是净损失。
\end{itemize}

净效果：高峰"提价多赚"被"驱离需求少赚"抵消，低谷"填谷多做量"被"低价薄利"抵消，厂家总利润近似不变。这与价格歧视的标准结论一致——\emph{供给固定、需求总量固定时，时间价格歧视主要重新分配剩余，不必然放大厂家利润。}

---

## 3. 关键追问：削峰省下的拥塞损失为何没变成利润

QoS 从 0.76 回升到 0.97，少折损大量请求，直觉上应是纯增益。两个原因使其未落到厂家利润：

\begin{enumerate}
  \item \textbf{削峰是靠驱离高峰需求实现的}。把利用率从 0.78 压到 0.66 必须挤走部分高峰用户；QoS 回升多结算的量，大致等于被挤走需求的损失。\emph{保 QoS 的代价恰是丢需求}——这正是 QoS 改善为真、利润却不涨的原因。
  \item \textbf{竞争与退出选项封住涨价空间}。异质双厂家竞争 + 直连/退出选项的存在，使任一厂家无法靠提价独吞 QoS 改善的剩余；涨价过度则需求流失。因此 QoS 改善的剩余更多流向\emph{用户}（体验改善、价格被竞争压制），而非厂家利润。
\end{enumerate}

QoS 从 0.756 大幅回升到 $0.97$--$0.99$（两种网格分辨率一致），而系统利润在更精细的求解下从 $+9.3\%$ 翻转为 $-1.2\%$——\textbf{利润变化的符号本身随求解精度翻转，表明真实利润效应接近于零、被求解误差淹没}。这从数值上佐证了第 2--3 节的机制论证：QoS 改善是稳健的真实效应，利润效应则近似中性。

> 注：细网格 FP \textbf{未收敛}——regret 在约第 10 轮触底于 $\approx14$ 后不降反升、40 轮后漂移到 $\approx22$（目标 <5），故 $-1.9\%$ 为未收敛快照、不可作定量结论；粗网格的 regret$=4.98$ 收敛部分是粗粒度假象。但"利润方向不稳健、QoS 改善方向（粗细网格一致）稳健"这一定性结论不受影响——这正是本节论点只依赖 QoS 方向、不依赖利润数值的原因。

## 5. 稳健性边界（诚实声明）

\begin{itemize}
  \item \textbf{稳健（分辨率无关）}：削峰（P2）、QoS 保护（P3）在粗/细网格下方向一致。
  \item \textbf{不稳健（须谨慎或撤回）}：利润增益的符号随网格翻转，故只能声称利润\emph{近似中性}，不可声称提升；"哪个厂家更依赖动态定价"（原 P4）方向亦随网格翻转，应撤回。
  \item \textbf{区间依赖}：弹性负载迁移强于刚性（P1）在非拥塞区稳健，在拥塞均衡处抹平。
  \item 全部经济参数为合成设定，具体百分比不作生产预测，仅支撑机制方向。
\end{itemize}

## 6. 政策与机制含义（论文卖点）

\begin{enumerate}
  \item \textbf{反直觉发现}：通常认为动态定价/surge pricing 是厂家趁拥塞牟利的工具；本文表明，在竞争 + 算力固定 + 用户可退出的推理服务市场，它\emph{主要是 QoS 保护工具，厂家并不因此显著多赚}。
  \item \textbf{监管含义}：监管者不必将分时定价视为牟利手段——其剩余主要流向服务质量与用户体验。
  \item \textbf{运营含义}：运营者采用分时定价的正当理由应是"保障 SLA/服务质量"，而非"提升收入"；把它当收入工具会落空。
\end{enumerate}
