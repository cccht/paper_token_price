# 计算机论文框架图图标来源与许可审计

检索日期：2026-07-10

## 1. 目的与边界

本次检索回答两个问题：计算机论文作者通常从哪里获取框架图图标，以及哪些来源适合当前论文的可编辑 Draw.io 图。检索以小红书和哔哩哔哩的实际绘图经验为线索，再到各素材库的官方网站核对许可。社交平台内容只用于观察常见工作流，不作为版权判断依据。

本轮未下载或复制社交平台中的图标，也未把商业素材嵌入论文。登录状态由有头 Chrome 的独立本地配置保存；没有读取、导出或记录 Cookie、访问令牌及其他凭据。

## 2. 社区中最常见的来源

检索结果没有显示一个公认的“SCI 计算机论文专用图标库”。作者通常从素材网站取得 SVG 或 PNG，再在 PowerPoint、Visio、Draw.io、Illustrator、Inkscape 或 Figma 中统一颜色、线宽和排版。

按本轮帖子中的出现频率和讨论强度，常见来源大致为：

1. Flaticon；
2. Iconfont；
3. Iconfinder；
4. IconPark；
5. Icons8 和 IconScout；
6. Streamline、Remix Icon、React Icons、Material Icons 等。

这一排序只说明社区使用频率，不表示许可更宽松，也不表示图标更适合正式投稿。

## 6. 对当前论文的建议

当前 Figure 1 使用 Font Awesome Free，许可和归属路径清楚，但图形语言偏单色界面符号。小红书和哔哩哔哩中更常见、也更接近用户期望的方案是 IconPark、Flaticon 或 IconScout。考虑到论文源文件会公开，三者不能等同处理。

建议按以下顺序改图：

1. 以 **IconPark** 作为唯一主图标家族，优先使用双色或多色主题；
2. 只在 IconPark 缺少关键语义时，少量补充 Tabler 或 Material 图标；
3. 物联网场景仅保留用户终端、API 网关、GPU 服务集群、QoS 监测和退出选项等必要图标；
4. 价格、流量和 QoS 由统一的线型、箭头和小标签表达，不再增加装饰性图标；
5. 若一定要使用等距或 3D 物联网素材，应原创绘制一小套，或购买并保存允许期刊发表及源文件交付的明确许可；
6. 在 `figure_sources/` 中保留图标名称、版本、原始链接、许可文本和修改记录。

因此，下一版 Draw.io 最稳妥的方案不是直接使用小红书帖子里的素材，而是采用社区同样常用、许可更清楚的 **IconPark**，重新统一用户、网络、服务器和监测图标。这样能提升物联网观感，同时保留论文资产的可编辑性和可公开复现性。

## 7. 不应采用的做法

- 不从论文截图、社交平台配图或素材网站预览图中裁剪图标；
- 不以“非商业用途”为由跳过许可核验；
- 不绕过下载限制或水印；
- 不把付费库存素材嵌入可公开提取的 Draw.io 源文件，除非许可明确允许；
- 不把不同透视、线宽、阴影和配色的多个图标库混在同一张框架图中；
- 不因图标丰富而增加论文模型中不存在的实体或数据流。
