## Reviewer 1

**Overall assessment**: 本文在推理服务定价文献中提出了一个有价值的建模视角——将中间商作为独立博弈主体纳入三层 Stackelberg 框架，并将 QoS 退化作为内生反馈。理论推导结构完整，实验消融设计较为周全。但技术严密性在若干关键环节存在缺口，尤其是数值实验结果的因果解释需要更谨慎的限定。

**Who would be interested and why**: 推理服务系统研究者会对 QoS 退化的博弈论建模感兴趣；平台经济学者关注多主体 Stackelberg 均衡计算方法；API 平台运营商可将其用作仿真工具。但定价决策者不会直接使用合成参数下的结果。

**Major strengths**:

1. 三层博弈中每一层的微观基础都有形式化处理（有限势博弈→logit 连续近似→中间商 KKT→平台逆向归纳）。
2. 消融设计干净：无 QoS 感知动态定价作为消融基线，直接分离"策略自由度"和"QoS 内化"两个效应。
3. 用户福祉实验诚实地暴露了系统利润提升与用户效用下降之间的张力。

**Major concerns**:

1. **条件性定理的实用性边界未充分讨论**：命题 3 的 SPE 存在性完全依赖假设 1。该假设要求中间商利润拟凹、Nash 均衡对应上半连续、SUE 响应连续——在含 logit 需求和 QoS 退化的实际利润函数中，这些条件极可能不成立。评注 1 虽然提及"全局范围内呈现非凸性"，但未给出跨参数范围的拟凹失效诊断。
2. **纯策略 Nash 均衡与连续 logit 响应之间的关系说明不充分**：命题 1 证明有限用户 PSNE 存在，但主数值实验使用的是大人口 logit 近似。虽然第 3.3.2 节引用了标准人口博弈近似，但这种过渡的可微性要求与 N=60→N=∞ 的收敛关系未做任何定量验证。
3. **需求函数的两段结构（刚性+sigmoid + 弹性+logit）** 在第 4.1 节中引入但从未在理论部分出现，这导致理论模型与数值实现之间存在重要 gap。命题 3 的证明建立在连续响应上，但 sigmoid 价格响应函数可能引入额外的非凸性。

**Technical failings that need to be addressed before the case is established**:

1. 必须量化有限 PSNE 的经验分布与 logit SUE 之间的差距（至少用 N=60 的 PSNE 枚举与 logit 聚合结果做一次对比）。
2. 假设 1 的拟凹条件至少需要画一张利润函数切面图（固定其他策略，扫描自身变量），证明在合成参数下该条件是否近似成立。
3. `ε-Nash regret` 定义为 solver-based regret（第 4.3 节仅校验 SLSQP 搜索范围内是否存在改进），需要明确区分于标准非合作博弈中的 ε-Nash 均衡。建议将"近似 Nash 均衡"改为"solver-suboptimality bounded candidate"或等效术语。

**Assessment against Nature-style criteria**:

|------|------|------|
| originality | 中高 | 将中间商策略自由度、QoS 反馈和三层 Stackelberg 结合是新的，但单平台-多中间商博弈结构在电力市场已有大量先例（作者引用了 Wang 2026），将此框架平移到推理服务的新意主要集中在退化函数的引入上 |

| interdisciplinary readership | 中 | 博弈论/运筹学社区和 ML 系统社区都可能关心，但各自的切入点差异较大 |

**Recommendation posture**: 潜在地支持，但在以下条件满足前不构成已确立的技术论据：(1) 假设 1 的数值验证；(2) 有限博弈与连续响应的定量衔接；(3) "Nash" 术语精度的修正。

## Reviewer 2

**Overall assessment**: 本文从建模新意和问题定位上是有趣的工作——推理服务的 QoS 退化定价确实缺乏完整博弈论建模。但声称的贡献幅度（系统利润提升 116%）远超出一般机制论文的合理区间，这提示要么合成参数设置巧合地大幅放大了效应，要么基线策略（统一零售 0.80）过于低效。读者需要被说服这 116% 的提升中有多少是定价差异化的贡献，有多少是容量优化单独能解释的。

**Who would be interested and why**: 多层定价博弈和 AI 经济学的交叉研究者。策略商人会关注"中间商→用户→QoS→利润"的反馈链是否在真实参数下成立。但真实市场中，OpenRouter 等聚合商的差价远比本文模型小（通常 5%–15%），零售价上下界 0.45–2.10 的设定需要更贴近实际报价数据。

**Major strengths**:

1. 将中间商从价格中转站提升为有策略自由度的博弈主体，这一步相比于 Yan 2026 等二层 Stackelberg 有明显推进。
2. 用户保护收入最大化实验表明，价格保护下平台收入目标与用户体验可以兼容——这一结果对实际部署有参考价值。
3. 作者在讨论中明确列出了 6 项限制，诚实度值得肯定。

**Major concerns**:

1. **基线选择的不对称性**：统一零售基线不仅固定价格，还固定容量分配比例。因此 116% 的系统利润提升混杂了至少三个正交效应：定价差异化、容量动态分配、以及 QoS 退化成本的内化。消融实验中固定容量使系统利润降低 7.43%，仅解释了小部分。更好的消融基线应是"仅优化容量不优化价格"和"仅优化价格不优化容量"，以系统分离各效应。
2. **vLLM 校准的强度不足以支撑"真实 QoS 剖面"**：只测了两个极小的开源模型（0.5B 和 3B）在某单张 RTX 4090 上的首 token 延迟达标率。首先，这与实际 API 聚合商请求的多模型/多硬件异构环境相差甚远；其次，首 token 延迟只是 QoS 的一个维度（吞吐、完成率、超时率都是）。拟合出的 κ=0.52 和 κ=0.94 远小于合成设定的 κ=15.0，但实验结果显示系统利润几乎不变——这暗示模型对 κ 不敏感（与 QoS 阈值敏感性结果一致），也暗示 QoS 退化成本 c_q=0.35 可能是主要的利润驱动因子，作者对此未做讨论。
3. **中间商利润大幅增长但用户效用下降**（inclusive value 2.4981→-0.0437）：这实质上是在"用户多付钱获得更好的 QoS"和"用户因为涨价流失"两种力量作用下的结果。负的 inclusive value 说明平均用户效用为负。这在离散选择模型中可以解释（涨价效应压倒了 QoS 改善），但作者未分析这是否意味着在真实支付意愿下用户将大量流失。

**Technical failings that need to be addressed before the case is established**:

1. 需要至少完成一次与真实 API 市场价格区间的对照：选择 DeepSeek-v3 或 GPT-4o 在 OpenRouter 上的实际批发/零售价差，代入模型看预测是否在合理范围内。
2. 需要分离三个效应（价格差异化、容量动态化、QoS 内化）对 116% 利润提升的相对贡献。
3. vLLM 校准的局限性需要大幅加强讨论，至少不应将两个 0.5B/3B 模型的测量称为"现实 QoS 校准"。

**Assessment against Nature-style criteria**:

| 维度 | 评级 | 说明 |
|------|------|------|
| originality | 中高 | 框架本身有创新，但核心博弈论工具（势博弈→logit→逆向归纳）都是标准件，创新在组合方式而非工具本身 |
| scientific importance | 中 | 对 AI 经济学和推理服务运营有潜在意义，但目前证据状态不足以支持"outstanding"判定 |
| interdisciplinary readership | 中高 | 博弈论、系统、ML 运营三个社区都可能关注，但都可能需要大量背景补充 |
| technical soundness | 中偏低 | 数值结果的幅度和因果归因都不够坚实 |
| readability for nonspecialists | 中 | 博弈论文本较密集，非本领域读者可能难以追踪推导链 |

**Recommendation posture**: 有兴趣但需要对效应幅度做更严密的因果归因，并在真实价格区间上做一次外部验证。

## Reviewer 3

**Overall assessment**: 文章在推理服务和定量博弈论的交叉领域提出了一个有意思的框架。三层模型结构清晰，消融和敏感性分析设计用心。然而，作为一篇面向跨学科读者的工作，该文在关键概念衔接上存在 gap——尤其是从有限势博弈到连续 logit 均衡、从全局最优到 solver-based regret、从"均衡"概念到"数值候选"之间的术语漂移。此外，文章用中文撰写且面向本地化期刊风格（ctexart），但其理论贡献和实验设定定位的是英语国际学术社区的对话者（大量引用 Nature/ICML/NeurIPS 类文献）。这一张力需要在目标读者群上澄清。

## Cross-review synthesis

**Consensus strengths**:
- 三层博弈结构（平台↔中间商↔用户+QoS 反馈）本身就是对现有二层 Stackelberg 建模的推进。
- 消融和鲁棒性分析的设计质量高于同类理论+仿真工作。
- 作者诚实地报告了局限性、regret 意义、以及系统利润/用户效用的矛盾。

**Consensus technical risks**:
- 假设 1（拟凹、上半连续、连续选择对应）在含 logit 需求和 QoS 退化的利润函数中是否近似成立，是全部定理推论的根基——三位审稿人均认为该验证未提供。
- 116% 的系统利润提升因基线过弱和效应混杂而被高估。
- 从有限 PSNE 到连续 logit SUE 的过渡缺乏定量验证。
- "Nash" 和 "均衡" 术语在与 SLSQP 类求解器搭配使用时需要加限定词。

**Where emphasis differs across reviewers**:
- **Reviewer 1** 最关注技术严密性（定理条件验证、PSNE→logit 衔接、Nash 术语精度）。
- **Reviewer 2** 最关注效应解释（116% 的因果归因、vLLM 校准覆盖、价格区间现实性）。
- **Reviewer 3** 最关注表述可读性（叙事结构、符号密度、跨学科友好性、贡献重心）。

**Broad-interest / significance readout**:
在合成参数下展示的机制方向（价格→容量→需求→QoS 反馈链）对推理服务经济学有概念贡献。但要从"概念演示"升级为"已确立的科学声明"，需要在真实价格区间和更广泛的 QoS 退化证据上获得外部验证。当前状态更适合作为一个有前景的模型提案和计算方法，而非已确立的经济命题。

**Most important issues to resolve before a strong Nature-style case is established**:

1. 证明或数值验证假设 1 的关键条件在合成参数下的近似成立程度。
2. 构造更好的消融基线，分离价格差异化 vs 容量动态化 vs QoS 退化内化对利润提升的相对贡献。
3. 使用至少一组真实 API 市场价格区间（如 OpenRouter 或 AWS Bedrock 的公开批发/零售价）做一次外部比对。
4. 将术语 "Nash 均衡" 在数值实验中改为更精确的限定性表述（如 "candidate equilibrium with solver-based regret ≤ 10⁻³"）。
5. 精简符号表，低使用频率的证明专用符号放入附录。
6. 修复随机效用模型中 τ_i 异质偏好→1-a_t 聚合代理的理论 gap。
