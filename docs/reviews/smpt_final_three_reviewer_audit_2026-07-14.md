## Reviewer 1：理论、博弈与数值证书

### 总体评价

论文已经给出完整的需求、渠道、路由、拥塞、QoS、利润和有限博弈链条。式 (1)--(18) 定义市场，式 (19)--(25) 给出混合 Nash 条件与 regret，命题 1 和命题 2 的结论与证明范围基本准确。与早期稿相比，当前稿没有再把纯策略快照或低分辨率网格误写为连续均衡。

### 主要优点

- 式 (5) 和式 (7) 明确保持 OD 质量与总需求守恒，峰值下降不会由退出或市场收缩机械产生。
- 命题 1 只证明固定点存在，不声称唯一性或迭代必然收敛。
- 788 候选有限博弈的全候选 regret 为 `1.14e-13`，并与 `0.400%/0.473%` 的非零离网格 regret 分开报告。
- 676 个活跃剖面的中间商独立搜索和 21,632 次固定点多初值检查提供了有用的数值 verification。

### 主要问题

1. **混合均衡分支恢复审计已完成。** 最终受限收益矩阵使用报告解、均匀初值和 64 个独立 Dirichlet 初值重复求解。66 个初值中 65 个返回有效解，并在 `1e-6` 概率容差下聚为同一分支；四项主指标与 baseline 一致到浮点误差。该结果支持报告分支的数值恢复稳定性，但不穷举或证明所有双矩阵均衡唯一。

2. **候选集具有审计自适应性。** Table 2 已公开 770 个确定性向量和 18 个延续向量，baseline 离网格检查也已通过。这足以支持当前有限博弈，但不是预先固定的全连续策略设计。

   修改方案：保留 `audit-adaptive finite candidate set` 的定位；不要把离网格门禁改写为连续均衡证明。对敏感性场景只允许声称共同候选集内稳健性。

3. **中间商策略类仍然受限。** 式 (18) 的连续优化覆盖三个参数，但不覆盖逐时段独立零售价和路由。`beta=0` 占 29.0% 概率质量，至少一个时段近确定性路由的剖面占 24.9%，说明响应形状常靠近简化策略类的边界。

   修改方案：保留边界质量报告和 `numerical routing control` 解释；不要使用 globally optimal response。当前 676 剖面 differential-evolution 审计约束的是中间商自身目标误差，不能单独证明下游服务商支付充分稳定。

4. **中间商目标误差不能直接约束服务商支付误差。** 2026-07-16 的只读后验检查分别用 stored response 和 independent differential-evolution candidate 重算 676 个正概率剖面。若每个剖面采用两者中中间商利润较高者，四项主运行指标的概率加权变化低于 `5e-6` 相对量级，A/B 期望收益变化为 `+0.01065/-0.00991`。但是原混合策略在这张修正后的 26-by-26 活跃支付矩阵上产生 A/B 活跃集 regret `1.1441/0.0102`，A 的相对值约 `0.137%`。主要变化来自概率质量仅 `0.0001867` 的剖面 `(625,112)`：中间商利润提高 `0.10268` 时，A/B 单剖面支付变化约为 `+57.13/-52.95`。这说明接近确定性路由时，中间商目标的近平台区间可以伴随服务商间较大的支付重分配。

   修改方案：自动 post-sensitivity 管线重建正式中间商审计后，生成带来源哈希的 follower-payoff sensitivity 工件。正文应报告加权主指标稳定，同时明确该诊断不覆盖全部偏离支付，因此不能把缓存矩阵上的 `1.14e-13` regret 外推为独立重优化响应下的同等 regret。除非全候选支付按更强 follower response 全量重建，否则均衡结论必须保留 `under the evaluated numerical payoffs`。

5. **统一价格基线的 12 个零斜率候选过粗。** 正式 off-grid 工件检查的是动态 788 候选混合均衡，没有检查统一价格受限博弈。2026-07-16 的完整临时二维基价搜索对两方各评估 918 个候选：服务商 A 的最佳偏离约为 `(0.5749,0,0.6136,0)`，利润由 `944.2488` 增至 `1054.1449`，相对 regret 为 `11.64%`；服务商 B 的最佳偏离约为 `(0.4978,0,0.7663,0)`，利润由 `875.1931` 增至 `918.3268`，相对 regret 为 `4.93%`。全部评估均收敛，最大联合残差低于 `1.0e-9`。该幅度远高于数值误差，说明当前 `(0.575,0,0.600,0)` 只是 12-candidate game 的纯均衡，不足以代表可信的统一价格基线。

   修改方案：保留已完成的动态 788 候选结果，但扩展零斜率共同候选集，对 baseline 与八个扰动分别重求统一有限博弈并执行二维离网格检查。确定性预设计使用旧 12 个锚点、seed `20260716` 的 512 点二维 Latin hypercube、`9x13` 全域格和两个半宽为完整定义域跨度 `5%` 的 `9x9` 局部格；新增唯一规则数为 `12+512+114+81+81=800`，与旧动态集合的并集预期为 1,576 条。重建前，`12.32%`、`15.70%`、`0.890→0.959` 及所有敏感性相对变化均不得作为投稿结论。临时搜索完成后应生成来源绑定的正式基线诊断，而不是引用 `/tmp` 数值。

### 推荐意见

当前为**大修**。动态混合均衡分支门禁已通过；还必须修复统一价格基线、完成八场景动态敏感性并重建全部比较后，理论与数值证据才可能达到 SMPT 机制仿真论文的可评审水平。

## Reviewer 2：实验设计、验证与现实性

### 总体评价

实验层级比通常的单场景机制仿真更透明。BurstGPT 只锚定负载形状，vLLM 只锚定 QoS 形状，效用价格系数、迁移成本、容量和成本被明确标为 synthetic design。结果表区分了剖面级极值期望与期望曲线极值，避免了混合策略下常见的统计量错置。当前稿已把 model verification 与 empirical input anchoring 分开，并明确市场层输出没有 external validation。

### 主要优点

- BurstGPT 使用 1,429,737 条固定版本记录，去掉首尾不完整日，并对 59 个完整日给出日间标准差。
- vLLM 报告硬件、软件、模型、并发点、请求数、prompt、输出上限、warm-up、seed、重复次数和 Student-t 区间。
- Figure 4 同时呈现有限候选 regret、离网格 regret、固定点残差和中间商响应检查。
- 机制分解与价格形状分解均明确为 fixed-policy accounting，不冒充重新求得的均衡反事实。

### 主要问题

1. **统一价格比较基线尚未充分求解。** 当前统一博弈只有 12 个零斜率策略。完整临时二维基价搜索已发现服务商 A/B 分别约 `11.64%/4.93%` 的相对收益改进，因此主表和摘要中的动态相对统一变化不是最终可投稿数字。

   修改方案：构造更密且可审计的共同零斜率候选集，使用 deviation expansion 求统一有限博弈，并对两家服务商执行独立二维离网格检查。baseline 与每个扰动场景都必须使用同一扩展设计。

2. **八组全量敏感性尚未完成。** 旧 788 候选阶段已有 6/8 个扰动通过场景、来源、regret 和固定点门禁；`migration_cost_low` 正在续算，`qos_threshold_low` 排队。Section 4.5 仍有 `FINAL_SENSITIVITY_RESULTS` 占位，且所有相对统一价格的幅度必须在 800 条统一规则上重算，因此当前不能声称容量、效用价格系数、迁移成本和 QoS 阈值扰动下方向稳定。

   修改方案：每个场景必须绑定 baseline SHA-256、788 候选、连续中间商响应、`1e-7` 有限 regret 和 `1e-8` 固定点残差门禁。只有全部通过后才生成汇总图和正文。

   设计边界：这些场景是局部单因素重求解。容量扰动保持 `G_A/G_B=2.5`，效用价格系数扰动保持两类用户的相对比例；刚性类型不可迁移，因此迁移成本缩放只改变弹性类型的计算流量。扰动场景不逐一执行 scenario-specific off-grid expansion，也不重复 baseline 676 个正概率剖面的 differential-evolution follower audit 和固定点多初值 audit。即使八场景全部通过，也只能说明共同 788 个规则内的局部方向，不能据此声称覆盖异质性比例变化、参数交互、连续策略邻域或全局参数空间。

3. **经济行为没有真实校准。** BurstGPT 不含价格实验，电力价格反应不能直接移植为推理用户弹性。当前 12.32% 峰值下降还受到统一基线离散化缺口影响，修复前不能作为机制幅度引用。

   修改方案：摘要、Section 4.6 和结论继续使用 `synthetic economic calibration`、`mechanism study` 与 `not a production forecast`。不要把与电力实验的量级比较写成外部验证。

4. **QoS 锚点的外推范围有限。** 测量只有单张 RTX 4090、两个较小 Qwen2.5 模型、一个短 prompt、128 输出 token 和五次重复。归一化并发不是物理 GPU utilisation。

   修改方案：正文已把 0.5 秒 TTFT 标为预设诊断阈值，并明确并发代理不等于硬件利用率。两个工件都记录最大模型长度 2,048；0.5B 运行日志保存完整服务端命令，3B 工件未嵌入完整命令，但同一实验时段的执行日志保存了 `--dtype auto`、`--max-model-len 2048` 和 `--gpu-memory-utilization 0.75`。后续若不新增测量，仍须保留单 GPU、短生成和小模型的外推限制，并说明 3B 的完整命令来自旁证日志而非研究工件字段。

5. **日间不确定性没有进入均衡。** Figure 2(a) 给出 59 天标准差，但均衡只使用平均日形状。

   修改方案：不应声称对日间随机负载稳健。可在未来工作中提出按日或 bootstrap 负载形状的随机仿真；本轮不以未执行分析支撑结论。

6. **超载由约化 QoS 曲线表示，而不是动态队列。** 模型允许 `u>1`，但没有服务积压、跨时段 backlog 或队列稳定性条件。主结果中的 `1.223` 是服务压力指标，不能解释为一个经验证可长期稳定运行的队列状态。

   修改方案：方法和局限性已明确 `G_m` 是固定服务率参考值，超载影响由 QoS 下降表示。保持该约化模型边界，不新增 queue-stability claim；若后续提高现实性，应将流量守恒与排队状态转移一起重求均衡。

### 推荐意见

当前为**大修**。统一价格基线存在明确的离散化错误，且敏感性工件尚未完成。只有扩展统一候选、重求比较并让全部场景通过门禁后，才能重新评估是否降为小修级别。

## Reviewer 3：贡献定位、结构与语言

### 总体评价

五章结构清楚，论文已不再像实验日志。Introduction 提出 temporal load shifting 与 spatial substitution 的识别问题；Related work 以电力定价为基础，再连接推理服务和有限博弈；Experimental results 按输入、主结果、数值检查、机制和限制展开。英文总体直接，AI 高频连接词、宣传性词汇和规则化三段排比很少。

### 主要优点

- 题目准确包含固定容量、time-of-use pricing、仿真和两类替代机制。
- 摘要在 250 词以内，包含目的、模型、数据锚点、求解、主要结果和结论边界。
- Table 1 直接区分四条文献线索的来源、本文增加内容和限制。
- Figure 1 的可编辑 Draw.io 源、节点逻辑、字体嵌入和页面可读性均已通过内部技术检查；但其布局生成器接受过 Codex 辅助，因此按 SMPT 当前图像政策只能作为内部审阅蓝图，不能直接上传。
- 关键 2025--2026 引用已通过 AAAI、NBER、arXiv、KDD 和作者/出版方记录核验；电力动态定价综述另加入两篇经 ScienceDirect 正式页面核验的 2024 年博弈论与强化学习工作。
- 所有正式图和表均已有正文交叉引用；Streamline 图标的 CC BY 4.0 署名、固定来源 commit 和本地许可文件齐全。

### 主要问题

1. **贡献可信但范围较窄。** 本文不是新均衡定理、新电价机制或生产调度器。可发表性来自 conserved OD modelling、空间替代分离和分层数值 verification 的组合。

   修改方案：保持当前三句式贡献逻辑，不增加“novel framework”“robust improvement”等宣传性表述。

2. **候选集细节较密。** Table 2 和 18 个延续向量增加了可复现性，也中断了方法叙事。

   修改方案：用户要求 PDF 自包含，因此保留这些值；通过前后两段解释其目的，避免使用“第四候选集”等内部版本术语。当前稿已完成该修改。

3. **投稿元数据尚不合规。** `Anonymous Author` 不符合 SMPT single-anonymized title page 要求；Funding、CRediT、competing interests、通讯作者地址和 ORCID 尚未提供。

   修改方案：作者提供真实信息后再生成投稿 title page 和声明。不得由工具推测或编造。

4. **最终模板尚未迁移。** 当前 generic article 适合内部审读，但正式提交应使用 Elsevier 官方 LaTeX 模板并按系统要求单独提交图文件、Highlights 和声明。

5. **投稿图像须由作者独立重建。** SMPT 明确禁止 generative AI 或 AI-assisted tools 创建或修改投稿图像。当前框架图布局和数据绘图脚本均曾接受 Codex 辅助；矢量格式、确定性绘图或人工复核不构成政策例外。

   修改方案：保留现有 Figures 1--6 作为内容与数值核对蓝图，由作者在不使用生成式或 AI 辅助图像工具的条件下独立重建，并记录逐图校验和确认信息。

### 推荐意见

当前为**大修**，但语言与结构本身已接近投稿稿。剩余工作主要是补齐已计划证据、投稿元数据和官方模板，而不是继续堆砌术语。

## Cross-review synthesis

### 共同认可

- 市场模型、质量守恒、利润会计和有限博弈条件能够相互对应。
- 动态 788 候选的有限 regret、离网格 regret、固定点和中间商审计均有机器可读证据；统一 12 候选基线没有同等级证据。
- 论文对 synthetic calibration、有限策略空间和非生产预测的边界表述较准确。
- QoS 改善证据强于利润改善证据；当前主案例明确报告利润下降。

### 共同阻塞项

1. 扩展零斜率统一价格候选集，对 baseline 与八个扰动重求统一有限博弈并完成二维离网格检查。

3. 删除敏感性占位，重建 Figure 6、结果段、摘要和结论数字边界。
